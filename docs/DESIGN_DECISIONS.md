# QuizCraft 设计决策文档

> 基于 5 轮 Exa Agent 竞品/技术研究 + 用户痛点深度访谈 (2026-06-23)

---

## 一、产品定位

**一句话**：开源自部署的自适应学习引擎——从文档自动出题、智能教学、间隔复习、终端实操，四位一体。

**与竞品的核心差异**：
- 不是"上传 -> 出题 -> 做题"的单向流（Quizlet/Knowt/Revisely 模式）
- 而是"诊断 -> 不懂就教 -> 懂了再考 -> 错了用你的文档解释 -> 间隔重复"的自适应闭环
- 错题反馈引用用户文档原文，不是通用解析
- 开源 + 用户自带模型，零供应商锁定

---

## 二、第一用户画像

- **身份**：大学生，学科跨度大（数学、计算机、通识等）
- **学习模式**：速成型，考前 3-7 天集中突击，平时不太学
- **现有习惯**：边读 PPT 边做笔记，但笔记做了不回看；会找真题/别人的题来做
- **三层遗忘**：看懂不等于记住、今天记明天忘、考场提取失败
- **核心痛苦**：做错题后搜解析（大学搜题酱等），看了还是不懂——因为解析不是基于自己学的内容写的
- **理解者偏差**：上过课有印象的内容看文档没问题，没上过/没听懂的必须先补课才能看懂文档
- **使用场景**：考前突击 + 日常学习新技能 + 面试准备

## 三、技术栈

| 层 | 选型 | 理由 |
|----|------|------|
| 前端 | React + Next.js | 生态最大，xterm.js 集成成熟 |
| 后端 | Python (FastAPI) | 文档解析/LLM SDK 生态最强 |
| 数据库 | SQLite | 零配置，单文件，备份=拷贝，适合开源自部署 |
| 部署 | Docker Compose 优先 | 一键启动；同时支持手动部署、Vercel+云服务器 |
| LLM | 用户自选/自带配置 | 可插拔：云端 API (Claude/GPT/Gemini) 或本地模型 (Ollama) |

## 四、核心功能设计决策

### 4.1 文档解析——分层路由

不用单一解析器，按页面复杂度自动升级：

| 层级 | 工具 | 处理场景 | 成本 |
|------|------|----------|------|
| L1 快速 | PyMuPDF4LLM | 干净数字 PDF 文本提取 | 几乎免费 |
| L2 结构 | Docling (推荐) | 复杂表格/公式/多栏/图文混排 | 本地 GPU |
| L3 视觉 | 多模态 LLM | 扫描件/手写/图表密集页 | ~$0.003-0.006/页 |
| DOCX | Pandoc / Mammoth | Word 文档 | 免费 |
| 公式 | Marker + Texify | LaTeX 公式识别提取 | 本地 GPU |

**关键原则**：300 页教材可能只有 30 页需要 L3，大部分走 L1 即可。

**分块策略**：结构感知章节分块，每块 512-1024 token，保留元数据（章节路径、页码、图表ID）。

### 4.2 出题引擎——两步生成法

**第一步：提取学习目标**
- LLM 从文档段落提取核心概念、易错点、关键定义
- 不直接出题，先建知识图谱

**第二步：按 Bloom 层级出题**
- 每个概念至少 2 道：记忆/理解 + 应用/分析
- 混合题型：选择题、简答题、填空题、判断题、实操题
- 完全可控参数：章节范围、难度、题型比例、数量

**Prompt 工程方案（研究验证）**：
1. 来源锚定：每题必须引用文档原文 source_span
2. 错误选项基于常见误解，不是随机错误
3. 自我批评管线：生成后模型自评，淘汰低分题
4. Few-shot 示例引导高阶题目生成
5. 学习目标优先：先识别核心概念再出题，避免出一堆定义背诵题

### 4.3 自适应学习引擎

**核心流程**：
1. 上传文档 -> 按章节/知识点拆解
2. 每个知识点出 2-3 道诊断题（简单）
3. 做对 -> 标记"已掌握"，进入 FSRS 间隔重复池
4. 做错 -> 进入教学模式：用文档原文解释概念，再出题
5. 交叉混合复习：不按文档顺序，混合新旧知识点

**错题反馈**：必须引用用户文档的具体段落，不是通用解释。
- 示例："你课件第 37 页的定义 X 说明了..."，而不是"X 的定义是..."

### 4.4 间隔重复——FSRS 算法

**选型修正**：从 SM-2 改为 FSRS

| 对比 | SM-2 | FSRS |
|------|------|------|
| 参数 | 固定/手调 | ML 自适应 21 参数 |
| 记忆率 | 无法设定 | 可设目标（如 90%） |
| 逾期处理 | 惩罚过重 | 平滑处理 |
| 状态 | 过时 | Anki 23.10+ 默认支持 |
| 开源库 | sm-2 (Python) | py-fsrs / ts-fsrs |

**实现**：后端 py-fsrs，前端 ts-fsrs。保留 SM-2 作兼容选项。

### 4.5 闪卡系统

- FSRS 驱动的间隔重复调度
- 从文档自动生成正反面闪卡
- 用户评分：记住 / 模糊 / 忘了 / 简单（映射 FSRS Again/Hard/Good/Easy）
- 归档机制：考完/掌握的内容一键归档，停止推送
- 支持 Anki .apkg 导出（可选，后续）

### 4.6 终端闯关

**前端**：xterm.js（唯一正确选择）
**后端**：WebSocket + ttyd 或 node-pty
**沙箱**：Docker 容器（非特权、--network none、cgroups、pids 限制、TTL 清理）
**验证**：KillerCoda 模式——LLM 生成题目时同时生成 check.sh，exit code 0 = 通过，非 0 给提示
**与文档联动**：解析 Linux 教材后自动生成实操题 + 验证脚本

### 4.7 学习看板 + 归档

- 看板视图：待学习 | 进行中 | 已掌握
- 可拖拽/手动调整状态
- 归档：一键归档已考完/掌握的课程，所有相关闪卡/题目停止推送
- 仪表盘主页展示学习状态概览

### 4.8 考前突击模式（竞品完全没有）

1. 上传文档 + 设置考试日期（如"3 天后考"）
2. 系统自动规划：
   - Day 1: 诊断 + 教学（定位不会的）
   - Day 2: 巩固 + 间隔复习（强化薄弱点）
   - Day 3: 模拟考试（混合交叉出题）
3. 跳过已掌握内容，不浪费突击时间
4. 考前最后一天：交叉混合出题模拟真实考试

## 五、数据模型核心

```
Document -> Sections -> Concepts -> Questions -> Reviews
                                       |
                                   Flashcards -> FSRS Schedule
                                       |
                                   TerminalChallenges -> check.sh
```

每个 Question 必须关联 source_span（文档原文位置），这是"用你的文档解释错题"的数据基础。

## 六、开发优先级

### Phase 1: 核心四件套（先让自己用起来）
1. 文件解析（PDF + docx，分层路由）
2. 智能出题（混合题型，Bloom 层级，来源锚定）
3. 答题 + 评分（选择题即时判分，简答题 LLM 评分，错题引用文档解释）
4. 闪卡系统（FSRS 间隔重复）

### Phase 2: 自适应 + 规划
5. 诊断 -> 教学 -> 测试 自适应流程
6. 考前突击模式（自动规划复习节奏）
7. 学习看板 + 归档机制

### Phase 3: 终端闯关
8. xterm.js + Docker 沙箱
9. 从教材自动生成实操题 + check.sh
10. 终端题与理论题联动

### Phase 4: 生态完善
11. 多文档关联（一门课多个 PPT + 教材，跨文档出题）
12. 多部署方式文档
13. 用户认证系统
14. Anki 导出兼容

## 七、补充决策（2026-06-23 第二轮）

### 7.1 多文档关联——课程文件夹模式

- 用户创建「课程」，往里放多个文件（PPT + 教材 + 笔记）
- 系统自动合并知识图谱，跨文档出题（如 PPT 里的概念 + 教材里的例题混合出一道应用题）
- 每道题仍标注来源文档 + 页码，可溯源
- 数据模型新增：`Course -> Documents[]`，Course 级别的 Concepts 去重合并

### 7.2 错题本——FSRS 自动调度 + 变体题生成

- **不做独立的"错题本"页面让用户去看**（用户不会看）
- 错题自动进入 FSRS 间隔重复池，算法更高频推送
- 错题列表可查看（作为参考，不是主要入口）
- 核心功能：一键生成「错题变体题」——不是原题重做，而是同一知识点的新题
- 这样用户不需要主动"看错题本"，系统自动安排复习 + 换题角度巩固

### 7.3 面试 vs 考试——场景标签切换

- 创建课程时选择场景：「考试」或「面试」
- 同一出题引擎，不同配置：
  - 考试模式：选择题/填空/判断为主，Bloom 偏记忆-应用层
  - 面试模式：开放简答/「用自己的话解释」/实操题为主，Bloom 偏分析-评价层
- 题型比例和评分标准不同，底层共享

### 7.4 离线使用——已有题目本地可用

- 出题、解析生成、简答评分 **必须联网**（需要 LLM）
- 已生成的题目、闪卡、FSRS 调度数据 **全部本地存储**（SQLite）
- 无网时可以：做题（客观题即时判分）、闪卡复习、查看错题解析
- 无网时不能：出新题、简答评分（联网后补判）
- 架构含义：前端需要本地缓存策略，不能每次都请求后端

### 7.5 用户认证——简单密码登录

- Phase 1：环境变量 `QUIZCRAFT_PASSWORD` 配置密码
- 单用户设计，无注册流程
- 登录后发 session token（httpOnly cookie）
- 后期可升级为完整用户系统（多用户 + OAuth）
- 自部署场景足够，不过度设计
