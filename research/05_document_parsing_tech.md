## PDF 解析器对比

| 工具 | 类型 | 表格 | 公式 | 图片 | 最适合 | 成本 |
|------|------|------|------|------|--------|------|
| PyMuPDF4LLM | 本地 OSS | 中 | 弱 | 好 | 干净数字 PDF 快速提取 | 免费 |
| pdfplumber | 本地 OSS | 好(需调参) | 弱 | 弱 | 调试/表格提取 | 免费 |
| pdf2image+OCR | 管线 | 差-中 | 差 | 好 | 扫描 PDF 兜底 | 免费(本地) |
| **Marker** | **ML OSS** | **好-很好** | **好(Texify)** | **好** | **教材/论文/STEM** | **本地GPU** |
| Unstructured.io | OSS+商业 | 好(hi_res) | 中 | 好(hi_res) | 生产 RAG 管线 | 免费/付费 |
| LlamaParse | 商业云 | 很好 | 中-好 | 好 | 最高开箱质量 | $1.25-56/1k页 |
| **Docling** | **MIT OSS** | **很好** | **好(改进中)** | **好** | **推荐默认** | **本地GPU** |
| Amazon Textract | 商业云 | 好 | 弱 | 好 | 企业OCR | $1.5-15/1k页 |
| Azure Doc Intelligence | 商业云 | 好 | 弱-中 | 好 | 企业文档 | $1.5-30/1k页 |

### 推荐方案: 分层路由

```
PDF 输入
  │
  ├─ L1: PyMuPDF4LLM (快速提取)
  │    └─ 成功且质量够 → 使用
  │    └─ 失败/质量差 → 升级
  │
  ├─ L2: Docling (结构化提取)
  │    └─ 处理表格/公式/多栏
  │    └─ 仍有问题 → 升级
  │
  └─ L3: 多模态 LLM (视觉理解)
       └─ 扫描件/手写/复杂图表
```

## DOCX 解析器对比

| 工具 | 优点 | 缺点 | 推荐用途 |

## 多模态 LLM 直接解析方案

- **方法**: 每页 PDF→图片(150-220 DPI)→送视觉模型→结构化 Markdown
- **何时用**: 扫描件/多栏+图表/合并单元格表格/数学/手写 — 不是默认方案
- **成本估算**: 
  - Claude Sonnet: ~$3.89-4.69/1k 张 1-3MP 图
  - GPT-4o: 数百到低千 image tokens/页
  - Gemini: 280-1120 tokens/页
  - 比文本提取贵 10-100x
- **推荐模型**: GPT-4.1/4o 或 Claude Sonnet 做质量；Gemini Flash 做低成本大批量
- **原则**: 混合方案——PyMuPDF/Docling 先跑，只把失败页送视觉模型

## 分块策略

| 策略 | 描述 | 适用 | 推荐 |
|------|------|------|------|
| **结构感知** | 按章节/标题切，512-1024 token，10-20% overlap | **教材/讲义/大多数文档** | **默认** |
| 固定递归 | 段落/句子边界切，固定窗口 | 简单文本，成本优先 | 备选 |
| 层级父子 | 小块检索，大块上下文 | 教材(既要精确又要解释) | Phase 2 |
| 语义 | 相邻句子相似度切 | 无标题的乱序笔记 | 特殊 |
| 上下文增强 | 给每块加 LLM 生成的上下文头 | 引用多/代词多的章节 | Phase 2 |

## 实用建议总结

1. **分层路由不用单一解析器**: PyMuPDF4LLM → Docling/Marker → LlamaParse/Azure/VLM
2. **保留结构元数据**: document_id, 章节路径, 页码, bbox, 表格/图/公式 ID, 标题
3. **混合分块**: 章节感知父块 + 512-1024 子块, 表格/公式原子保留
4. **按页面类型计费**: 
   - 纯文本页 ≈ 免费
   - OCR/表格页 ≈ $1.5-15/1k 页
   - 公式页 ≈ $5/1k 页
   - 视觉解析 ≈ $3-50+/1k 页
5. **在自己的语料上评测**: 取 50-100 页代表性文档，比较阅读顺序/表格/公式/图片/下游出题质量
